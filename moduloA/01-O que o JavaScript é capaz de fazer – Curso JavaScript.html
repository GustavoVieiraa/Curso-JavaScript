Client x Server

Vamos começar, entendendo o conceito de cliente x servidor.

Para utilizar alguns serviços como Instagram, facebook, gmail, realizamos solicitações para um servidor, onde estão hospedados esses serviços, utilizando nossos dados como cliente.

Para acessar o Instagram, realizamos o acesso pelo nosso navegador por exemplo, pela URL, ele vai para a infraestrutura da internet em um servidor DNS e direciona diretamente para o serviço que solicitamos.

Esse servidor, ele é parecido com a nossa maquina, ele tem um sistema operacional rodando nele, com pastas e diretorios com arquivos. Esse arquivo é o que nós clientes, estamos interessados, ele pode ser uma foto, um texto, uma vídeo.

Na maioria das vezes, é um documento HTML, como esse que estamos desenvolvendo, o servidor vai gerar uma cópia desse HTML e vai mandar para mim (cliente), no meu Google chroome, ou navegador padrão. A partir dai, o navegador vai trabalhar para gerar o nosso site, carregando nosso HTML, CSS, JavaScript, PHP e etc.

Mas, basicamente, temos tecnologias para Clientes e Tecnologias para Servidores.

Quais tecnologias, vamos precisar conhecer para o desenvolvimento?

HTML5, CSS3 e JavaScript.

  • HTML5 - Conteúdo do Site, parte Semântica.
  • CSS3 - Toda estilização do site!
  • JavaScript - Toda parte lógica e programada do site.
